SVGs are a scalable image format, which means they will easily scale to any size and retain their quality without increasing their filesize.

“SVG” stands for “Scalable Vector Graphics”

SVGs are often used for:

  • Icons
  • Graphs/Charts
  • Large, simple images
  • Patterned backgrounds
  • Applying effects to other elements via SVG filters

SVGs are defined using XML and not HTML: Extensible Markup Language (HTML-like syntax which is used for lots of things, from APIs, to RSS, to spreadsheet and word editor software.)

  1. Absolute Units
  2. pixel "px": absolute one as size remains same throughout the website and not relative to any other element
  3. inch (in) and centimeters (cm) are other absolute units
  4. Relative Units:

    Relative units are units that can change based on their context. There are several of them that you are likely to encounter and want to use.

  5. em and rem both refer to a font size, though they are often used to define other sizes in CSS. You’ll see both of them often so we’re going to explain both, but as a rule-of-thumb, prefer rem.
  6. 1em is the font-size of an element (or the element’s parent if you’re using it to set font-size). So, for example, if an element’s font-size is 16px, then setting its width to 4em would make its width 64px (16 * 4 == 64).
  8. 1rem is the font-size of the root element (either :root or html). The math works the same with rem as it did with em, but without the added complexity of keeping track of the parent’s font size. Relying on em could mean that a particular size could change if the context changes, which is very likely not the behavior you want.
  10. Viewport units
  11. The units vh and vw relate to the size of the viewport. Specifically, 1vh is equal to 1% of the viewport height and 1vw is equal to 1% of the viewport width. These can be useful any time you want something to be sized relative to the viewport, examples including full-height heroes, full-screen app-like interfaces.
